D’après le W3C, le SVG (abréviation de «Scalable Vector Graphics») est un format de fichier qui permet à des applications graphiques en deux dimensions de s’afficher en XML. Il permet trois types d’objet graphiques : des formes vectorielles (traits, courbes, ellipses..), des images et du texte. Contrairement aux formats GIF et JPEG qui ont une taille spécifique, les documents SVG peuvent être agrandis sans perte de résolution et être modifiés directement dans le HTML et le CSS.

1. LES FORMES VECTORIELLES SIMPLES.

Les formes vectorielles simples en format SVG peuvent être directement intégrées dans le code HTML dans des balises <svg>.

Pour le rectangle, il faut ouvrir une balise (ex :<rect>) et y inclure :
- svg width et svg height, qui correspondent à la largeur et la hauteur du canevas SVG (soit la zone dans laquelle le SVG apparaît).
- «rect x» et «rect y» qui correspondent à la position du rectangle x et en y;
- «rect width» et «rect height» qui correspondent à la largeur et la hauteur du rectangle;
La balise «style» quand à elle permet de contrôler le style du SVG directement dans le html. Ici,
- «fill» correspond à la couleur du fond;
- «stroke» correspond à la couleur du contour;
- «stroke-width» correspond à l’épaisseur du contour;
- «fill-opacity» correspond à l’opacité du fond;
- «stroke-opacity» correspond à l’opacité du contour; 

					

						
html


						
						
<svg width="220" height="200">
<rect x="50" y="50" width="100" height="100" style="fill:grey;stroke:black;stroke-width:3;fill-opacity:0.1;stroke-opacity:0.9">  
</svg>

Pour le cercle, il faut ouvrir une balise (ex: ) et y inclure :
- «cx» et «cy», qui sont les positions du cercle en x et en y;
- «rx» et «ry», qui correspondent au rayon du cercle en x et en y. Dans cet exemple, le style du cercle a été appliqué dans le css en ouvrant une classe «ellipse». 

					

						
html


						
						
<svg height="220" width="220">
<ellipse class="ellipse" cx="110" cy="100" rx="50" ry="50" />
</svg>

Pour le triangle, il faut ouvrir une balise (ex: <path>), dont les coordonnées de l’exemple correspondent à :
- M90, position point du haut en x;
- 50, position point du haut en y;
- L35, position point à gauche en x;
- 50, position point à gauche en y;
- L150, position point à droite en x;
- 150, position point à droite en y. 

					

						
html


						
						
<svg height="220" width="220">
<path d="M90 50 L30 150 L150 150 Z"
style="fill:white;stroke:grey;stroke-width:3" />
</svg>

2. IMPORTER DU SVG

Il existe différentes manières d’importer un SVG sur un éditeur de texte. Les trois éditeurs graphique les plus populaires sont Illustrator, Sketch et Inkscape.

La première consiste à, comme représenté avec l’étoile, copier/coller le code SVG sur un éditeur de texte. Pour récupérer ce code sur Illustrator : fichier/ enregistrer sous, sélectionner SVG / cliquer sur code SVG... / copier coller le code affiché dans votre document HTML. 

					

						
html


						
						

<svg width="200px" height="200px" viewBox="0 0 200 200" enable-background="new 0 0 200 200" xml:space="preserve">
<polygon fill="#9D9D9C" stroke="#000000" stroke-width="3" stroke-miterlimit="10" points="100.146,17.235 120.69,80.462 187.171,80.462 133.387,119.537 153.931,182.765 100.146,143.689 46.363,182.765 66.906,119.537 13.123,80.462 79.603,80.462 "/>
</svg>

La seconde consiste à, comme représenté avec l’écureuil, l’enregistrer en format SVG et l’importer dans des balises <img src>. Pour enregistrer un SVG sur illustrator : fichier/ enregistrer sous, selectionner SVG / OK / enregistrer dans le dossier img de votre page web/ insérer img src sur le html. Il est alors recommandé de modifier directement le style du SVG dans le document SVG lui même à l’intérieur de balises <style>. 

					

						
html


						
						
<img src="img/ecureuil.svg" alt="ecureuil">

La troisième manière, qui représente les deux cercles de l’exemple, permet surtout de s’organiser et se repérer plus facilement et rapidement dans les codes du SVG. En effet, sur illustrator, si les différentes formes d’une même image sont rangées sur différents calques, le code SVG se modifie légèrement : des balises <g id=«Calque»> sont crées. Ils est ainsi plus pratique de modifier le css pour chacun des calques lorsque l’on veut que chacun d’entre eux ai un style différent.

3. LES CONTOURS

L’apparence du contour est définie par les attributs de couleur (stroke), de largeur (stroke-width) et/ou de pointillés (stroke-dasharray).

Les 3 rectangles du premier exemple possèdent des propriétés CSS différentes: le premier est noir, avec un épaisseur de 2px et a comme longueur de pointillés 1px; le deuxième est noir, avec un épaisseur de 2px et a comme longueur de pointillés 5px; le troisième est aussi noir, avec un épaisseur de 3px et a comme longueur de pointillés 30px. 

					

						
html


						
						
<svg width="200px" height="200px" viewBox="0 0 200 200" enable-background="new 0 0 200 200" xml:space="preserve">
<g id="rectangle3">
<g>
<rect x="21.649" y="144.279" fill="#C6C6C6" width="156.701" height="38.414"/>
</g>
</g>
<g id="rectangle2">
<g>
<rect x="21.649" y="80.793" fill="#C6C6C6" width="156.701" height="38.414"/>
</g>
</g>
<g id="rectangle1">
<g>
<rect x="21.649" y="18.307" fill="#C6C6C6" width="156.701" height="38.414"/>
</g>
</g>
</svg>
						
					


					

						
css


						
						
#rectangle3{
  stroke:black;
    stroke-dasharray: 30;
    stroke-width:3;
}

#rectangle2{
  stroke:black;
  stroke-dasharray: 5;
  stroke-width:2;
}

#rectangle1{
  stroke:black;
  stroke-dasharray: 1;
  stroke-width:2;
}

Il existe aussi une propriété qui permet de spécifier l’apparence des extrémités d’une ligne. Il s’agit de la propriété stroke-linecap. Elle peut prendre trois valeurs :
- butt : c’est la valeur par défaut : l’extrémité est un carré;
- round : l’extrémité est arrondie.
Il est aussi possible d’animer les pointillés (ou «stroke-dasharray»). La forme doit impérativement avoir une épaisseur de trait pour que l’animation fonctionne.

Pour l’exemple du carré, l’attribut «stroke-dashoffset» a été utilisé. Il permet de décaler le démarrage des tirets petit à petit : c’est ce qui donne cette impression d’animation.
«5s» correspondent à la durée de l’animation (ou plus concrètement dans cet exemple, le temps que met un tiret pourrevenir à sa position initiale), et «linear infinite» fait en sorte que l’animation ne s’arrête jamais. 

					

						
html


						
						
<svg width="200px" height="200px" viewBox="0 0 200 200" enable-background="new 0 0 200 200" xml:space="preserve">
<g id="rectangle4">
<g>
<rect x="20" y="20" fill="#C6C6C6" width="150" height="160" style="fill:white;stroke:grey;
stroke-width:4"/>
</g>
</g>
</svg>
						
					


					

						
css


						
						
#rectangle4{
  stroke-dasharray: 100;
  -webkit-animation: dash 5s linear infinite;
}

@-webkit-keyframes dash {
  to {
    stroke-dashoffset: 1000;
  }
}

Pour l’exemple du polygone, l’attribut «stroke-dashoffset» a là aussi été utilisé. Ici, l’animation est définie à l’intérieur des fichiers SVG (recommandation du w3c) entre des balises <style>. Dans cet exemple, l’idée est de remplir le contour au fur et à mesure. On créé donc un tiret qui fait 0px au début de l’animation et 800px à la fin. Plus le tracé ou le contour est long, plus il faut augmenter le nombre de pixel final. 

					

						
html


						
						
<img src="img/losange.svg" alt="losange">
						
					


					

						
SVG


						
						
<defs>
<style>
#losange{
stroke-dasharray: 800;
stroke-dashoffset: 0;
-webkit-animation: dash 5s linear infinite;
-o-animation: dash 5s linear infinite;
-ms-animation: dash 5s linear infinite;
-moz-animation: dash 5s linear infinite;
animation: dash 5s linear infinite;
}

@-webkit-keyframes dash {
from {
stroke-dashoffset: 800;
}
to {
stroke-dashoffset: 0;
}
			
</style>

</defs>

Il faut aussi faire attention à bien utiliser les préfixes devant «animations» car ils permettent l’affichage des animations en fonction des différents navigateurs. Il est d’ailleurs préférable de tous les mettre, afin qu’il soient visible pour tout le monde:
-webkit-animation ;
-o-animation ;
-ms-animation ;
-moz-animation ;
- animation.

4. ANIMER DU SVG

Il est aussi possible d’animer la taille, la couleur ou la position du SVG.

Pour le premier exemple, c’est la taille du cercle qui est modifiée. Dans le HTML, il faut introduire des balises <animate>
- r correspond au rayon du cercle;
- from=»0» indique que le rayon du cercle au début de l’animation est de 0px;
- to=»60» indique que le rayon du cercle à la fin de l’animation est de 60px;
- dur=»0.4s» indique que l’animation dure 0.4 secondes;
- repeatCount=»indefinite» fait en sorte que l’animation se répète indéfiniment. 

					

						
html


						
						
<svg height="200" width="200" viewBox="0 0 200 200">
<circle cx="75" cy="75" r="20" style="fill:white;stroke:black;stroke-width:10">
<animate attributeName="r" from="0" to="60" dur="0.4s" repeatCount="indefinite"></animate>
</svg>

Pour le deuxième exemple, c’est la taille et la couleur du cercle qui sont modifiées. Il faut une balise ‹animate› pour chaque attribut d’animation. Pour que le cercle passe de petit à grand et de grand à petit, il faut définir une sorte de boucle. Avec l’attribut «begin» vous pouvez déclencher une animation à un instant précis ou lorsqu’une autre animation se termine.

Pour chaque changement de couleur, «begin»=«5s» indique le temps au bout duquel la couleur doit changer, ici, 5 secondes, attributeName= «fill» indique l’attribut que l’on veut modifier (ici c’est la couleur de fond du cercle que l’on veut modifier), fill=«freeze» le changement d’état (on aurait utilisé «remove» si l’on voulait enlever un effet); to=«black» dur=«1s» correspond au temps que prend l’animation à changer de couleur. 

					

						
html


						
						
<svg height="200" width="200">
<circle cx="100" cy="80" r="20">
<animate id="grand" begin="0s;petit.end" attributeName="r" from="50" to="75" dur="1s">
</animate>
<animate id="petit" begin="grand.end" attributeName="r" from="75" to="50" dur="1s">
</animate>
<animate begin="5s" attributeName="fill" fill="freeze" to="grey" dur="1s">
</animate>
<animate begin="10s" attributeName="fill" fill="freeze" to="black" dur="1s">
</animate>
<animate begin="15s" attributeName="fill" fill="freeze" to="grey" dur="1s">
</animate>
<animate begin="20s" attributeName="fill" fill="freeze" to="black" dur="1s">
</animate>

</circle>
</svg>

Pour le troisième exemple, c’est la position du cercle qui est modifiée. Dans le HTML, il faut introduire des balises <animateMotion>.

C’est après «path=» que l’on indique les différentes coordonnées par lequel le cercle doit passer. Ici, le cercle se repositionne 4 fois :

La position 1 est représentée par les coordonnées suivantes:
M12 indique la position cercle (soit lorsqu’il est sur la gauche de l’ecran) en x
75 indique la position cercle (soit lorsqu’il est sur la gauche de l’ecran) en y

La position 2 est représentée par les coordonnées:
L75 indique la position cercle (soit lorsqu’il est sur le haut de l’ecran) en x
12 indique la position cercle (soit lorsqu’il est sur le haut de l’ecran) en y

La position 3 est représentée par les coordonnées:
L137 indique la position cercle (soit lorsqu’il est sur la droite de l’ecran) en x
75 indique la position cercle (soit lorsqu’il est sur la droite de l’ecran) en y

La position 4 est représentée par les coordonnées:
L75 indique la position cercle (soit lorsqu’il est sur le bas de l’ecran) en x
137 indique la position cercle (soit lorsqu’il est sur le bas de l’ecran) en y

dur=«2s» indique que l’animation dure 2 secondes.
repeatCount=«ndefinite» fait en sorte que l’animation se répète indéfiniment. 

					

						
html


						
						
<svg width="150" height="150" viewBox="0 0 150 150">
<circle cx="0" cy="0" r="15">
<animateMotion path="M12 75 L75 12 L137 75 L75 137 Z" dur="2s" repeatCount="indefinite"></animateMotion>
</circle>
</svg>
