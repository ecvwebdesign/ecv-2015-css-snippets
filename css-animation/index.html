CSS Animation

CSS Animations est un module qui permet l'animation de la plupart des éléments HTML en utilisant CSS, sans JavaScript ou Flash. Les animations sont fluides, même avec un système plutôt chargé. Elles sont faciles à utiliser pour faire les animations simples.

L'animation avec le CSS peut avoir quelques limites, mais, globalement, elles offrent un certain nombre d'avantages par rapport aux animations qui sont enclenchées par JS. Plus faciles à mettre en place et plus rapides sont juste deux des avantages des CSS animations. Naturellement, les animations actionnées par CSS ont quelques inconvénients, il est par exemple assez difficile d’avoir un contrôle total dessus.

[ Passez le souris au dessus le carré pour voir l'animation. ]

1. Deux composants d'une séquence Animation CSS.

1. Animation Properties: les attributs de style décrivant l'animation.
( Associés à l'élément que vous souhaitez animer à l'aide de la propriété animation et de ses sous-propriétés. )

  • animation-name
  • animation-duration
  • animation-timing-function
  • animation-delay
  • animation-iteration-count
  • animation-direction
  • animation-fill-mode
  • animation-play-state
					

						
css


						
						
/* décrire les propriétés individuels */
{
	animation: bounce;
	animation-duration: 4s;
	animation-iteration-count: 10;
	animation-direction: alternate;
	animation-timing-function: ease-out;
	animation-fill-mode: forwards;
	animation-delay: 2s;
}

/* ou décrire les propriétés ensembles
animation: [animation-name] [animation-duration] [animation-timing-function] [animation-delay][animation-iteration-count] [animation-direction] [animation-fill-mode] [animation-play-state] */
{
	animation: bounce 4s ease-out 2s 10 alternate forwards;
}

2. @keyframes, qui définit l’animation grâce à des points de passage (keyframes)intermédiaires en utilisant un pourcentage. 0% indique le début de la séquence, tandis que 100% indique l'instant final de l'animation. 

					

						
css


						
						
/* @keyframes [animation-name] {
	from { propriété_à_animer: valeur_initiale; }
	to { propriété_à_animer: valeur_initiale; }
} */

@keyframes bounce {
	from { background:white; }
	to { background:yellow; }
}

"from" et "to" équivalent à 0% et 100%. On passe ainsi d'un état initial qui est 0% à un état final qui est 100% avec, si l'on utilise les pourcentages, différentes étapes intermédiaires. A chaque étape, on redéfinit les valeurs des styles CSS animées, la transition entre les valeurs étant réalisée automatiquement. 

					

						
css


						
						
/* @keyframes [animation-name] {
	0% { propriété_à_animer: valeur_initiale; } 
	45% { propriété_à_animer: valeur_intermédiaire; } 
	100% { propriété_à_animer: valeur_finale; }
} */

@keyframes bounce {
	0% { background:white; } 
	45% { background:black; } 
	100% { background:yellow; }
}
						

					

				
					

						
html


						
						
<html>
	<head>
		<div class="animation1">
		</div>
		<div class="animation2">
		</div>
		...
	</head>
	...
</html>
						
					


					

						
css


						
						
/* commence animer par hover */
.animation1:hover {
	 animation: coleur 3s infinite;
}

@keyframes coleur	{
	 from { background: #3eb1a9; 
		 opacity:.3; }
	 to { background: #e89213; 
		 opacity:1; }
}

.animation2:hover {
	 animation: coleurs 3s infinite;}

@keyframes coleurs	{
	 0% {opacity:.6; 
		 background: #3eb1a9;}
	 45% {opacity:.1; 
		background: #ca1a62;}
	 100% {opacity:1; 
		 background: #e89213;}
}

2. LES LIENS AVEC D’AUTRES PROPRIÉTÉS CSS

CSS Animations travaille tout simplement sur les mêmes valeurs que CSS Transition et Transformation.
Aussi CSS Animation peut combiner avec les propriétés des objets SVG pour animer directement des éléments SVG dans les navigateurs.

[ Attention!! : La différence principale entre les animations CSS et les transitions CSS réside dans la règle @keyframes. Par conséquent, vous êtes obligé d’exécuter l’animation dans une direction (du début à la fin) avec les animations CSS, toutes les propriétés voulues étant animées en même temps. ]

					

						
css


						
						

@keyframes bounceOutUp {
	20% { transform: translate3d(0, -10px, 0); }
	40%, 45% { opacity: 1;
			transform: translate3d(0, 20px, 0); }
	100% { opacity: 0;
		transform: translate3d(0, -100px, 0); }
}

@keyframes coleurs	{
	 0% { transform: scale3d(.3, .3, .3);
		 background: #3eb1a9; }
	 45% { transform: scale3d(.9, .9, .9);
		background: #ca1a62; }
	 100% { transform: scale3d(1, 1, 1);
		 background: #e89213; }
}

@keyframes tossing {
	0% { transform: rotate(-10deg); 
		background: #3eb1a9; }
	50% { transform: rotate(10deg);
		 background: #ca1a62; }
	100% { transform: rotate(-10deg); 
		background: #e89213; }			 
}
						

					

				
					

						
css


						
						
@keyframes bounce {
	0%, 20%, 53%, 80%, 100% { transform: translate3d(0,0,0); }
	40%, 43% { transform: translate3d(0, -8px, 0); }
	70% { transform: translate3d(0, -8px, 0); }
	90% { transform: translate3d(0,-2px,0); }
}

3.COMPATIBILITÉ DANS LES NAVIGATEURS

Les animations CSS3 sont compatibles avec IE 1.1, Firefox 3.6, Chrome 4.1, Safari 8, Opéra 2.7 et sur iOS Safari 8.1 et Android 4. Cependant, pour pouvoir les utiliser, il faut préfixer toutes les propriétés ainsi que les étapes définies avec @keyframes.

  • -webkit- : @keyframes et animation pour chrome,safari,opera.
  • -moz- : @keyframes et animation pour firefox.
  • -o- : @keyframes et animation pour opera 12.

Il existe d’autres conditions utilisables dans des cas plus rares, les lignes citées ci-dessus étant les plus couramment utilisées pour l'animation CSS.

					

						
css


						
						
.div {
	-webkit-animation: bounceOut;
		-moz-animation: bounceOut;
			animation: bounceOut;
}

@-webkit-keyframes bounceOut {
	0% { -webkit-transform: scale3d(.3,.3,.3);
		 	-moz-transform: scale3d(.3,.3,.3);
				transform: scale3d(.3,.3,.3); }
	100% {-webkit-transform: scale3d(1, 1, 1);
			moz-transform: scale3d(1, 1, 1);
				transform: scale3d(1, 1, 1); }
}

4. fonctionnalités de propriétés

Animation-Duration :
Définit la durée d’un seul cycle de votre animation - en secondes (s) ou millisecondes (ms).

Animation-Timing-Function :
Spécifie les valeurs des propriétés intermédiaires à utiliser pendant un cycle unique d’une animation.

Les options :

  • ease
  • linear
  • ease-in
  • ease-out
  • ease-in-out
  • initial
  • inherit

 Si vous voulez des options de timing plus avancées, vous pouvez créer vous-mêmes une courbe timing-function grâce à cubic-bezier curve.

Animation-Iteration-Count :
Configure le nombre de fois que l'animation doit se répéter : vous pouvez définir infinite pour que l'animation se répète indéfiniment.

Les valeurs :

  • #: un nombre spécifique d'itérations(1 par défaut).
  • infinite: l'animation se répète à l’infini.
  • initial: définit le nombre d'itérations à la valeur par défaut.
  • inherit: hérite de la valeur du parent.

Animation-Delay :
Configure le délai entre l'instant auquel l'élément est chargé et le début de l'animation.
La value est définie en secondes (s) ou millisecondes (ms).

					

						
css


						
						
/* même option de .div peut écrire ensemble */
/* nom-animation temps-duration iteration-count */
.div1,.div2,.div3,.div4,.div5 {
	position: relative;
	animation: mmove 1.5s infinite;
}

/* nom-animation temps-duration delay iteration-count */
.div6,.div7,.div8,.div9,.div10 {
	position:relative;
	animation: mmoves 1.5s 1s infinite;
}

/* timing-fonction */
.div1,.div6{ 
	animation-timing-function: linear;
}
.div2,.div7{ 
	animation-timing-function: ease;
}
.div3,.div8{ 
	animation-timing-function: ease-in;
}
.div4,.div9{ 
	animation-timing-function: ease-out;
}
.div5,.div10{ 
	animation-timing-function: ease-in-out;
}

@keyframes mmove {
	from { top: 0px; 
	to { top: 160px; }
}

@keyframes mmoves {
	from { top: 0px; }
	to { top: 160px; }
}

Animation-Direction :
Détermine si l'animation doit se jouer tour à tour en avant puis en arrière à chaque cycle de la séquence, ou si elle doit revenir à son point de départ et se répéter.

Les valeurs :

  • normal
  • reverse
  • alternate
  • alternate-reverse

Animation-Fill-Mode :
Configure les valeurs qui doivent être appliquées par l'animation avant et après qu'elle soit exécutée.

Les options :

  • backwards
  • forwards
  • both
  • normal

Animation-Play-State :
Vous permet de mettre en pause et de reprendre la lecture de la séquence animée.

Les valeurs :

  • playing
  • paused
					

						
css


						
						
.div1 {
	animation: rise 20s ease-in-out 2s infinite Alternate both;
}
.div2 {
	animation: slideOver 10s infinite Alternate-reverse both;
}

@keyframes rise {
	0% { background: tomato; opacity: 0; transform: translateY(50px); }
	10% { opacity: 1; }
	20% { background: goldenred; }
	55% { background: golden; }
	100% { transform: translateY(-50px); }
}

@keyframes slideOver {
	0% { transform: translateX(-150px); opacity: 0; }
	20% { opacity: 1; }
	90% { opacity: 1; }
	100% { transform: translateX(10px); opacity: 0; }
}

.div3:hover {
	cursor: pointer;
	animation: scale 2s forwards;
}
.div4:hover {
	cursor: pointer;
	animation: scale 2s;
}

5. Animations multiples

Pour ajouter des animations multiples à un sélecteur, séparez simplement les valeurs avec une virgule.

					

						
css


						
						
.div {
	animation: slideIn 2s, rotate 1.75s;
}
