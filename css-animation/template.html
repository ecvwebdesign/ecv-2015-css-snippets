CSS Animation

CSS Animations est un module qui permet l'animation de la plupart des éléments HTML en utilisant CSS sans JavaScript ou Flash. Les animations sont fluides, même avec un système plutôt chargé. Elles sont faciles à utiliser pour faire les animations simples.

Un site web responsive est un site internet dont l’apparence et la structure sont adaptés à toutes résolutions, de manière fluide et dynamique. Si autrefois, la tendance était à l’élaboration d’une version des sites internet dédiée aux appareils portables complètement indépendante du site web classique, il existe désormais des moyens faciles de rendre votre site adaptable sans devoir le dupliquer.

1. Deux composants d'une séquence Animation CSS.

1. Animation Properties: Un style décrivant l'animation
(L'élément que vous souhaitez animer à l'aide de la propriété animation et de ses sous-propriétés).

• animation-name
• animation-duration
• animation-timing-function
• animation-delay
• animation-iteration-count
• animation-direction
• animation-fill-mode
• animation-play-state

					

						
css


						
						
/*décrire les propriétés individuels*/
{animation: bounce;
 animation-duration: 4s;
 animation-iteration-count: 10;
 animation-direction: alternate;
 animation-timing-function: ease-out;
 animation-fill-mode: forwards;
 animation-delay: 2s;}

/*ou décrire les propriétés ensembles
animation: [animation-name] [animation-duration] [animation-timing-function] [animation-delay][animation-iteration-count] [animation-direction] [animation-fill-mode] [animation-play-state]*/
{animation: bounce 4s ease-out 2s 10 alternate forwards;}

2. @keyframes qui indique l’instant dans la séquence à laquelle elle est située du style CSS de l'animation, ainsi que des points de passage intermédiaires en utilisant un percentage. 0% indique le premier instant de la séquence, tandis que 100% indique l'instant final de l'animation. 

					

						
css


						
						
/*@keyframes [animation-name] 
{from {propriété_à_animer: valeur_initiale;}to {propriété_à_animer: valeur_initiale;}}*/
@keyframes bounce
{from {background:white;} to{background:yellow;}}

<"from" et "to" équivalent à 0% et 100%. On passe ainsi d'un état initial qui est 0% à un état final qui est 100% avec si l'on utilise les pourcentages différentes étapes intermédiares. A chaque étape, on redéfini les valeurs des styles css qui sont animés la transition entre les valeurs étant réalisée automatiquement.>

					

						
css


						
						
/*@keyframes [animation-name]
{ 0% {propriété_à_animer: valeur_initiale;} 
45% {propriété_à_animer: valeur_intermédiaire;} 
100% {propriété_à_animer: valeur_finale;}}*/
@keyframes bounce
{ 0% {background:white;} 
45% {background:black;} 
100% {background:yellow;}}
                        

					

				
					

						
html


						
						
<html>
	<head>
	  <div class="animation1">
	  </div>
	  <div class="animation2">
	  </div>
		...
	</head>
	...
</html>
						
					


					

						
css


						
						
/*commence animer par hover*/
.animation1:hover {
   animation: coleur 3s infinite;}

@keyframes coleur  {
   from {background: #3eb1a9; 
         opacity:.3;}
   to {background: #e89213; 
       opacity:1;}
}

.animation2:hover {
   animation: coleurs 3s infinite;}

@keyframes coleurs  {
   0% {opacity:.6; 
       background: #3eb1a9;}
   45% {opacity:.1; 
        background: #ca1a62;}
   100% {opacity:1; 
         background: #e89213;}
}

2. L’accointance avec les autres CSS

CSS Animations travaille tout simplement sur les mêmes valeurs que CSS Transition et Transformation..
Aussi CSS Animation peut combiner avec les propriétés des objets SVG pour animer directement sur des éléments SVG dans des navigateurs.

[Attention!! : La différence principale entre les animations CSS et les transitions CSS réside dans la règle @keyframes. Par conséquent, vous êtes obligé d’exécuter l’animation dans une direction (du début à la fin) avec les animations CSS aussi bien à animer toutes les propriétés voulues en même temps.]

					

						
css


						
						

@keyframes bounceOutUp {
  20% {transform: translate3d(0, -10px, 0);}
  40%, 45% {opacity: 1;
            transform: translate3d(0, 20px, 0);}
  100% {opacity: 0;
        transform: translate3d(0, -100px, 0);}
}

@keyframes coleurs  {
   0% {transform: scale3d(.3, .3, .3);
       background: #3eb1a9;}
   45% {transform: scale3d(.9, .9, .9);
        background: #ca1a62;}
   100% {transform: scale3d(1, 1, 1);
         background: #e89213;}
}

@keyframes tossing {
  0% {transform: rotate(-10deg); 
      background: #3eb1a9;}
  50% {transform: rotate(10deg);
       background: #ca1a62;}
  100% {transform: rotate(-10deg); 
        background: #e89213;}           
}
						

					

				
					

						
css


						
						
@keyframes bounce {
  0%, 20%, 53%, 80%, 100% {transform: translate3d(0,0,0);}

  40%, 43% {transform: translate3d(0, -8px, 0);}

  70% {transform: translate3d(0, -8px, 0);}

  90% {transform: translate3d(0,-2px,0);}
}

3.compatibilité dans les navigateurs

Les animations CSS3 sont compatibles avec IE 1.1, Firefox 3.6, Chrome 4.1, Safari 8, Opéra 2.7 et sur iOS Safari 8.1 et Android 4. Cependant, pour pouvoir les utiliser, il faut préfixer toutes les propriétés ainsi que les étapes définies avec @keyframes.

  • -webkit- : @keyframes et animation pour chrome,safari,opera.
  • -moz- : @keyframes et animation pour firefox.
  • -o- : @keyframes et animation pour opera 12.

Il existe d’autres conditions utilisables dans des cas plus rares, les lignes citées ci-dessus étant les plus couramment utilisées pour l'animation css.

4. utilisation de plusieurs feuilles de style

En cas de media queries lourds en styles CSS, on peut également séparer notre feuille de style en plusieurs feuilles de styles différentes, chacune étant associée à une résolution différente et contenant les styles attribués.

Il suffira de lier chaque feuille de style entre les balises <head> et </head> de votre fichier HTML, de la même manière d’une feuille de style CSS unique. La seule différence sera un attribut, différent pour chaque feuille de style liée, qui associera ces feuilles à leurs résolutions d’écran correspondantes :

					

						
html


						
						
<link rel="stylesheet" media="screen and (min-width: 500px) and (max-width: 900px)" href="style1.css" />

5. images flexibles

S’il est possible de rendre des images flexibles en utilisant l’attribut CSS max-width : 100%, cet élément pose encore des soucis d’affichage sur des écrans type Retina, mais manque surtout de pertinence quant au changement de format des médias. En effet, une image au format paysage s’affichera parfaitement sur un site web, mais s’en trouvera excessivement réduite pour correspondre à la largeur d’un smartphone.

On peut gérer ce changement de plateforme grâce à la balise <picture>, qui permet alors d’appeler différentes versions d’une même image (cadrée différemment, par exemple) selon la résolution de l’écran sur lequel elle est visionnée.

L’ordre d’appel des différentes versions de l’image est important : il faudra appeler les images de la résolution la plus grande jusqu’à la plus petite, pour éviter les conflits entre media queries.

					

						
html


						
						
<picture>
	<source srcset="image-1.jpg" media="(min-width: 1000px)">
	<source srcset="image-2.jpg" media="(min-width: 500px)">
	<img src="image.jpg" alt="description de l’image">
</picture>

On n’oublie évidemment pas de rajouter la balise viewport entre les balises <head></head> de notre page HTML afin d’assurer le fonctionnement des media queries.

6. texte flexible

Si l’on peut, comme avec les images, donner à son texte des tailles différentes en fonction de la résolution grâce aux media queries CSS, il est également désormais possible d’avoir du texte à la taille relative aux dimensions du viewport.

Il s’agit ici d’utiliser les attributs CSS “vw” et “vh”, acronymes de “viewport width” et “viewport height”. Une unité vw/vh correspond ici à 1% de la largeur ou de la hauteur de la fenêtre (selon l’unité choisie). Par exemple, pour une fenêtre large de 1000 pixels et haute de 500 pixels, 1 vw correspondra à une taille de texte de 10 pixels, et 1 vh correspondra à une taille de 5 pixels.

N’oubliez tout de même pas d’ajouter des tailles de texte “classiques”, en pixels, dans votre CSS afin d’assurer la visibilité de votre site en cas de navigateur incompatible.
