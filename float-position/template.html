Les propriétés de positionnement CSS vous permettent de positionner un élément. Il peut également placer un élément derrière un autre, et de spécifier ce qui doit se produire lorsque le contenu d’un élément est trop long. Nous allons apprendre à modifier la position des éléments sur notre page. Vous allez voir, il existe plusieurs techniques permettant d’effectuer la mise en page du site. Chacune a ses avantages et ses défauts, ce sera à vous de sélectionner celle qui vous semble la meilleure selon votre cas.

1. Le positionnement static.

Les éléments peuvent être positionnés en utilisant le ‘top’, le ‘bottom’, le ‘left’, et le ‘right’ propriétés. Cependant, ces propriétés ne fonctionnera que si la propriété position est réglé en premier. Ils travaillent également différemment selon le procédé de positionnement.

Il existe quatre méthodes de positionnement différents. On va d'abord parler le positionnement 'static' qui est le positionnement par défaut, il n'est pas affecté par le ‘top’, le ‘bottom’, le ‘left’, et le ‘right’ propriétés.

					

						
html


						
						
<html>
	...
	<body>
		<h1>Hello world!
		</h1>
	</body>
</html>
						
					


					

						
css


						
						
h1{
	position: static;
}

2. à chaque breakpoint ses styles css

Chaque résolution est ensuite définie pour chaque breakpoints par des intervalles de valeurs, indiquant les largeurs d’écran maximales et minimales d’affichage des styles concernés.

					

						
css


						
						
@media screen and (min-width: 900px) {
	body {
		color: red;
	}
}
						

					

				
					

						
css


						
						
@media screen and (min-width: 500px) and (max-width: 900px) {
	body {
		background-color: black;
	}
}

3. syntaxe

Comme aperçu plus haut, on peut donc attribuer des styles à des résolutions selon différentes conditions. Voici un exemple des différentes conditions que vous pouvez utiliser pour définir vos breakpoints :

  • @media screen all : toutes résolutions
  • @media screen and (min-width: Xpx) : écrans de largeur minimale de X pixels
  • @media screen and (max-width: Xpx) : écrans de largeur maximale de X pixels
  • @media screen and (min-width: Xpx) and (max-width: Ypx) : écrans de largeur minimale de X pixels et de largeur maximale de Y pixels

Il existe d’autres conditions utilisables dans des cas plus rares, les lignes citées ci-dessus étant les plus couramment utilisées pour le responsive web design.

4. utilisation de plusieurs feuilles de style

En cas de media queries lourds en styles CSS, on peut également séparer notre feuille de style en plusieurs feuilles de styles différentes, chacune étant associée à une résolution différente et contenant les styles attribués.

Il suffira de lier chaque feuille de style entre les balises <head> et </head> de votre fichier HTML, de la même manière d’une feuille de style CSS unique. La seule différence sera un attribut, différent pour chaque feuille de style liée, qui associera ces feuilles à leurs résolutions d’écran correspondantes :

					

						
html


						
						
<link rel="stylesheet" media="screen and (min-width: 500px) and (max-width: 900px)" href="style1.css" />

5. images flexibles

S’il est possible de rendre des images flexibles en utilisant l’attribut CSS max-width : 100%, cet élément pose encore des soucis d’affichage sur des écrans type Retina, mais manque surtout de pertinence quant au changement de format des médias. En effet, une image au format paysage s’affichera parfaitement sur un site web, mais s’en trouvera excessivement réduite pour correspondre à la largeur d’un smartphone.

On peut gérer ce changement de plateforme grâce à la balise <picture>, qui permet alors d’appeler différentes versions d’une même image (cadrée différemment, par exemple) selon la résolution de l’écran sur lequel elle est visionnée.

L’ordre d’appel des différentes versions de l’image est important : il faudra appeler les images de la résolution la plus grande jusqu’à la plus petite, pour éviter les conflits entre media queries.

					

						
html


						
						
<picture>
	<source srcset="image-1.jpg" media="(min-width: 1000px)">
	<source srcset="image-2.jpg" media="(min-width: 500px)">
	<img src="image.jpg" alt="description de l’image">
</picture>

On n’oublie évidemment pas de rajouter la balise viewport entre les balises <head></head> de notre page HTML afin d’assurer le fonctionnement des media queries.

6. texte flexible

Si l’on peut, comme avec les images, donner à son texte des tailles différentes en fonction de la résolution grâce aux media queries CSS, il est également désormais possible d’avoir du texte à la taille relative aux dimensions du viewport.

Il s’agit ici d’utiliser les attributs CSS “vw” et “vh”, acronymes de “viewport width” et “viewport height”. Une unité vw/vh correspond ici à 1% de la largeur ou de la hauteur de la fenêtre (selon l’unité choisie). Par exemple, pour une fenêtre large de 1000 pixels et haute de 500 pixels, 1 vw correspondra à une taille de texte de 10 pixels, et 1 vh correspondra à une taille de 5 pixels.

N’oubliez tout de même pas d’ajouter des tailles de texte “classiques”, en pixels, dans votre CSS afin d’assurer la visibilité de votre site en cas de navigateur incompatible.
