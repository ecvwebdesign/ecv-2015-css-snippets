l'atelier flexbox

Flexbox est un modèle de boîte flexible qui s’affranchit totalement des valeurs fixes. Très utile dans un contexte de page Web adaptive (mobile & moniteur).

tutoriel

Flexbox n'est pas nouveau, mais n'est pas encore un standard. Le fruit est meilleur quand il est cueilli à maturité, et pour les standards, c'est pareil. Je vous invite à suivre ces petits ateliers pour ceux qui veulent expérimenter ou bien changer leur manière de construire une page Web. Juré, c'est accessible aux newbies. Bon jeu.

1. syntaxe.

Avant toute chose, voici la syntaxe si particulière de flexbox. Elle est dense, mais elle va changer la donne quant à la construction d’une mise en page.
On appellera flex container, l’élément du HTML renfermant les boîtes.
On appellera flex items, les éléments flexibles ou boîtes flexibles.


					

						
css


						
display: flex || inline-flex
/* affichage d'un élément */

flex-direction : column || column-reverse || row || row-reverse || inherit || initial
/* l'axe de disposition des flex items. Appliquée au flex container */

flex-wrap: nowrap || wrap || wrap-reverse || inherit || initial
/* détermine si les flex items reviennent à la ligne. Appliquée au flex container */

justify-content: flex-start || flex-end || center || space-between || space-around || inherit || initial
/* l'origine des flex items dans le flex container. Appliquée au flex container */

align-items: flex-start || flex-end || center || baseline || stretch || inherit || initial
/* l'emplacement du groupe entier des flex items dans le flex container. Appliquée au flex container */

order: Z=(∞;..;-2;-1;0;1;2;..;∞)
/* détermine un ordre spécifique aux flex items, peu importe l'ordre dans le HTML. Appliquée aux flex items */

flex: flex-grow | flex-shrink | flex-basis
/* la largeur flexible des flex items. Appliquée aux flex items */

Quand aux vendor prefixes:


					

						
css


						
/* l’affichage */
display: flex
/* par défaut */

display: -webkit-box;
/* Chrome, Safari, Opera */

display: -moz-box
/* Firefox */

display: -ms-flexbox
/* IE */

display: -webkit-flex
/* Chrome, Safari, Opera */

/* la flexibilité des flex items */
flex:  (valeur)
/* par défaut */

-webkit-box-flex: (valeur)
/* Chrome, Safari, Opera */

-moz-box-flex:  (valeur)
/* Firefox */

-webkit-flex:  (valeur)
/* Chrome, Safari, Opera */

-ms-flex:  (valeur)
/* IE */

/* l’ordres des flex items */
order: (valeur)
/* par défaut */

-webkit-box-ordinal-group: (valeur)
/* Chrome, Safari, Opera */

-moz-box-ordinal-group: (valeur)
/* Firefox */

/* Chrome, Safari, Opera */

2. flexbox construit des listes

					

						
html


						
<…>
<ul class="flexContainer">

	<li class="flexItem"><p>Patates</p></li>
	<li class="flexItem"><p>Papier toilettes</p></li>
	<li class="flexItem"><p>2 éponges</p></li>
	<li class="flexItem"><p>Ravioli</p></li>
	<li class="flexItem"><p>Canard WC</p></li>
	<li class="flexItem"><p>Slip de bain</p></li>
	<li class="flexItem"><p>Tartines</p></li>
	<li class="flexItem"><p>Sopalin</p></li>

</ul>
<…>
						
					


					

						
css


						
/* l’élément parent */
. flexContainer {
	/* déclarer le comportement au flex container */
	display: flex;

	/* modifier la direction de l’axe de haut en bas */
	flex-direction: column;

	/* les flex items reviendront à la ligne automatiquement */
	flex-wrap: wrap;

	/* reset de l’élément ul */
	padding: 0;
	list-style: none;
	max-width: 960px;
	margin: 0 auto;
}

/* Les éléments à l’intérieur des flex items */
.flexItem > p {
	padding: 15px;
}

Flexbox construit des systèmes de lignes et de colonnes. Prenons l'exemple de cette liste de courses très banale, gérable en quelques lignes de CSS. Dans un premier temps, il faut donner un comportement flexible au conteneur: le flex container (cf syntaxe).

Le conteneur est fixé. Maintenant c'est au tour des boîtes à l’intérieur de se comporter comme élément flexible: les flex items. On veut une liste sur une colonne, et pour cela, il faut que l'axe principal parte du haut vers le bas. Les flex items prendront automatiquement la largeur du flex container grâce à la propriété flex-basis et flex-shrink.

3. flexbox imbrique votre navigation.

					

						
html


						
<…>
<ul class="flexContainer">

	<li class="flexItem"><a href="#">accueil</a></li>
	<li class="flexItem "><a href="#">projets</a></li>
	<li class="flexItem "><a href="#">à propos</a></li>
	<li class="flexItem "><a href="#">contact</a></li>

</ul>
<…>
						
					


					

						
css


						
/* l’élément parent */
. flexContainer {
	/* déclarer le comportement au flex container */
	display: flex;

	/* modifier la direction de l’axe de gauche en droite */
	flex-direction: row;

	/* les flex items reviendront à la ligne automatiquement */
	flex-wrap: wrap;
	/* l’origine des flex items sera à l’extrémité droite plutôt qu'à gauche */
	justify-content: flex-end;

	/* reset de l’élément ul */
	padding: 0;
	list-style: none;
	max-width: 960px;
	margin: 0 auto;
}

/* Les éléments à l’intérieur des flex items */
.flexItem > p {
	padding: 25px 15px;
}

Nouveau cas de figure, les colonnes. Un peu plus compliqué car de nouvelles propriétés s'appliquent. Ce système permet de construire des grilles pour des pages web adaptées à son contenu.

Dans un premier temps, il faut inverser l'axe principal pour aligner les flex items. La propriété flex-direction s'applique au flex container.

Une fois alignés, il faut placer le groupe des flex items dans l'espace du flex container. La propriété justify-content s'applique au flex container.

Pour construire la barre de l'exemple, on doit placer les flex items à droite du flex container.
La longueur et la hauteur des flex items sont gérées par le padding attribué au contenu de ces items. La barre de navigation est construite, et en quelques propriétés seulement. Rien de bien méchant, mais ça va se corser à la prochaine étape.

4. flexbox et masonry font bon ménage

					

						
html


						
<…>
<ul class="flexContainer">

	<li class="flexItems mediumC">
		<div class="wrapper">
			<img src="img/test4.jpg" alt=" image à incorporer " />
			<span class="animation"><p>Bottle</p><button class="animation" alt="simuler un lien">jetez un œil</button></span>
		</div>
	</li>
	<…>
	<li class="flexItems smallC">
		<div class="wrapper">
			<img src="img/test5.jpg" alt="image à incorporer" />
			<span class="animation"><p>Geometric</p><button class="animation" alt="simuler un lien">jetez un œil</button></span>
		</div>
	</li>


</ul>
<…>
						
					


					

						
css


						
/* l’élément parent */
. flexContainer {
	/* déclarer le comportement au flex container */
	display: flex;

	/* modifier la direction de l’axede haut en bas */
	flex-direction: column;

	/* les flex items reviendront à la ligne automatiquement */
	flex-wrap: wrap;

	/* reset de l’élément ul */
	padding: 0;
	list-style: none;
	max-width: 960px;
	margin: 0 auto;
	/* dans ce cas-là, il faut spécifier une hauteur absolue */
	height: 2100px
}

/* On indique la taille que prendront les flex items avec la class smallC */
.flexContainer > .smallC {
	/* ces flex items prendront 1/3 de la largeur du parent */
	width : 33.33%
}

/* On indique la taille que prendront les flex items avec la class mediumC */
.flexContainer > .mediumC {
	/* ces flex items prendront 2/3 de la largeur du parent */
	width : 66.66%
}

/* l'interieur adaptra sa largeur en fonction du parent */
.flexItems > .wrapper, .flexItems > .wrapper img {
	width : 100% ;
}

Le niveau monte. Quand un jour on a besoin de créer une mise en page de grille dynamique, on fait appel généralement au principal plugin qui gère ça: Masonry. Le hic : Masonry est un plugin utilisant la librairie JQuery. Ce qui veut dire qu'il y a du Javascript. Le but est de montrer que cette grille peut se faire sans une seule once de JS.

Pour cela, commençons par donner un comportement au flex container, de manière à ce que les flex items forment une liste verticale.

À ce stade, les flex items prennent 100% de la taille du flex container. L'objectif est de faire remonter en haut quelques flex items. Il est alors essentiel de donner une valeur absolue à la propriété height, donnée au flex container.

Il y a un souci... Les flex items débordent du flex container. Dans ce cas-là, on procède alors à ce que fait Masonry: des largeurs différentes, pour former une mosaïque. On peut appliquer n'importe quelle taille, mais dans cette exemple, il y en a seulement 2. La propriété width s'applique aux flex items.

Cette manière de construire des grilles n'a pas vocation à remplacer Masonry, ce plugin gère mieux la mosaïque, pour un rendu propre. Flexbox en donne un avant-goût et permet de construire des grilles modulables rien qu'avec du CSS et un peu d'huile de coude, une bonne solution pour les néophytes.

La dernière étape vous attend en bas, c'est un peu le c-c-c-c-combo des capacités de Flexbox.

5. flexbox et le responsive

					

						
html


						
<…>
<ul class="flexContainer">

	<li class="flexItems sysCol8">
		<div class="wrapper">
			<h3>Je suis le 1e billet.</h3>
			<p>Avec pas mal de contenu au premier abord, car je suis pas mal large. Ce qui laisse pas mal d'écriture.</p>
		</div>
	</li>
	<…>
	<li class="flexItems sysCol6">
		<div class="wrapper">
			<h3>Pareil ce 9e billet.</h3>
			<p>Pareil chez le voisin.</p>
		</div>
	</li>

</ul>
<…>
						
					


					

						
css


						
/* l’élément parent */
. flexContainer {
	/* déclarer le comportement au flex container */
	display: flex;

	/* modifier la direction de l’axe de gauche à droite */
	flex-direction: row;

	/* les flex items reviendront à la ligne automatiquement */
	flex-wrap: wrap;

	/* reset de l’élément ul */
	padding: 0;
	list-style: none;
	max-width: 960px;
	margin: 0 auto;
}

/* On modélise le système de colonage, et celles-ci sont divisées en 12 colonnes */
.flexContainer > .sysCol1 {
	/* en utilisant la propriété flex */
	flex: 0 1 8.33%
}
.flexContainer > .sysCol2 {
	flex: 0 1 16.66%
}
.flexContainer > .sysCol3 {
	flex: 0 1 25%
}
.flexContainer > .sysCol4 {
	flex: 0 1 33.33%
}
.flexContainer > .sysCol5 {
	flex: 0 1 41.16%
}
.flexContainer > .sysCol6 {
	flex: 0 1 50%
}
.flexContainer > .sysCol7 {
	flex: 0 1 58.33%
}
.flexContainer > .sysCol8 {
	flex: 0 1 66.66%
}
.flexContainer > .sysCol9 {
	flex: 0 1 75%
}
.flexContainer > .sysCol10 {
	flex: 0 1 83.33%
}
.flexContainer > .sysCol11 {
	flex: 0 1 91.66%
}
.flexContainer > .sysCol12 {
	flex: 0 1 100%
}

/* On modélise les billets en appliquant une gouttière et une marge interne */
.flexContainer > .flexItems {
	padding: 25px;
	margin: 15px;
	background: #fff;
}

/* l’enveloppe aura la largeur du flex items, tout en suivant la marge interne */
.flexItems > .wrapper {
width : 100% ;
}

/* on fabrique un point de rupture à partir de 720px ou 45em */
@media only screen and (max-width: 45em) {
	
	/* et on y applique une seule et même propriété à nos colonnes */
	.sysCol1, .sysCol2, .sysCol3, .sysCol4,
	.sysCol5, .sysCol6, .sysCol7, .sysCol8,
	.sysCol9, .sysCol10, .sysCol11, .sysCol12 {
		flex: 0 1 100%;
	}

}

Le plus dur arrive, mais en réalité ce n'est pas si compliqué. Il s'agit dans un premier temps de dire au flex container de construire une liste en colonne, où les éléments peuvent revenir à la ligne.

Ensuite, pour faire varier la taille des boîtes, il suffit de constituer un système de grille avec différentes valeurs de flex basis.

En appliquant ces différentes tailles aux flex items, ceux-ci auront une taille différente, afin de créer un rythme. Pour le responsive il vaudrait mieux, avant toute chose, consulter le tutoriel d’Elodie sur le responsive web design. Il faudrait dire au CSS via @media screen qu'au point de rupture (breakpoint), flex basis prendra une valeur de 100%, par rapport au flex container.

En redimensionnant, vous apercevrez que les boîtes vont se redimensionner et s'étirer, tout en conservant une harmonie dans le flex container.
